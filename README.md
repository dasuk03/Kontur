# Контур — полевое обследование КТП

Веб-прототип Telegram WebApp для обследования КТП, фидеров, опор и приборов учета.

## Что уже работает

- карта/схема маршрута с объектами и выбором опоры;
- интеграция с Яндекс.Картами через `VITE_YANDEX_MAPS_API_KEY`;
- демонстрационный режим без API-ключа;
- карточка КТП: трансформатор, мощность, ПУ, ТТ, коэффициент трансформации, уличное освещение;
- карточка опоры: количество ног, дом на участке, фонари и один/несколько ПУ;
- чек-лист по сверке схемы, ПУ, фазности, шунтам, сторонним подключениям и фотофиксации;
- загрузка фотографий с камеры телефона;
- определение GPS-позиции браузером;
- локальный экспорт обследования в JSON;
- экспорт текущей сети в Excel: лист «Лист осмотра» с колонками split/не split, GPS, фазностью, опорой, ИСУЭ и опросом;
- второй лист Excel «Структура сети» с цепочкой КТП → фидер → опора → ПУ;
- создание структуры обследования для одного оператора: населённый пункт → КТП → фидеры → опоры → участки/ПУ;
- экран «Схема населённого пункта» с иерархией объектов и автоматическими расхождениями;
- редактор электрической схемы в стиле Visio на SVG-холсте;
- автокомпоновка схемы: КТП → ввод → фидеры → опоры → дома/участки → ПУ → фонари;
- условные обозначения трансформатора, опоры, дома, ПУ, фонаря, ответвления, аппарата и замечания;
- ручное перемещение объектов, редактирование подписей/цветов/координат и добавление новых элементов;
- режим «Соединить» для ручного создания ответвлений между любыми элементами;
- экспорт схемы в SVG с размером страницы A3 и печать через системный диалог браузера;
- редактирование исходных данных и добавление фидеров/опор без изменения кода;
- автоматическое сохранение проекта в браузере и импорт резервной копии JSON;
- локальный реестр всех ТП из `dasuk03.github.io/RossetiIMDS/map.html`;
- импорт исходного `map.html` и JSON в реестр с объединением по ID;
- поиск по ID, названию, типу, статусу, ПУ и координатам;
- фильтры по исходному статусу и состоянию обследования;
- карточка ТП с паспортными данными, координатами, заметками и переходом в рабочий проект;
- понятный онбординг при первом запуске.
- корректное базовое поведение внутри Telegram WebApp через `Telegram.WebApp`.
- быстрый полевой ввод ПУ: готовые шаблоны для НАРТИС/Меркурий/СЕ 102, копирование GPS опоры и голосовой ввод номера или замечания;
- горячие клавиши `/` и `Ctrl/Cmd + K` для поиска, а `Ctrl/Cmd + Enter` — для сохранения карточки опоры;
- оптимизированная загрузка Excel-модуля: библиотека XLSX загружается только при нажатии кнопки экспорта;
- оптимизированное обновление Яндекс.Карт, реестра и редактора схемы без лишнего пересоздания тяжёлых элементов.

## Запуск

```bash
npm install
npm run dev
```

Для реальной карты Яндекс создайте `.env` на основе `.env.example` и укажите ключ:

```env
VITE_YANDEX_MAPS_API_KEY=ваш_ключ
```

Без ключа приложение запускается с локальной интерактивной схемой трассы, что удобно для демонстрации интерфейса.

## Как работать в единоличном режиме

1. Откройте «Настройки проекта».
2. Заполните населённый пункт, дату, паспорт КТП и все отходящие фидеры.
3. Добавьте по одной записи на каждую опору: выберите фидер, укажите номер опоры и добавьте один или несколько ПУ. Для каждого ПУ отдельно заполните split/не split, фазность 1ф/3ф, GPS домовладения, ИСУЭ и опрос.
4. В карточке ПУ нажмите подходящий «Быстрый шаблон», продиктуйте номер кнопкой `●` или вставьте его вручную. Кнопка «GPS опоры» переносит координаты опоры в конкретный ПУ.
5. Нажмите «Сохранить локально» или используйте `Ctrl/Cmd + Enter`.
6. В разделе «Схема населённого пункта» проверьте собранную иерархию и автоматически найденные расхождения.
7. Кнопка «Excel» выгружает рабочую таблицу для передачи в Excel. Кнопка «JSON»/«Импорт» используется как резервная копия проекта.
8. Откройте «Редактор схемы Visio». Система построит исходную схему по введённым объектам.
9. Перетащите элементы мышью для компоновки, выберите объект и исправьте подписи в правой панели.
10. Для ручного ответвления включите «Соединить», выберите первый и второй объект.
11. Нажмите «Скачать SVG» для получения редактируемого векторного файла или «Печать» для печати на A3.

## Реестр всех ТП

В разделе «Реестр всех ТП» находится локальная база из 2 180 объектов, перенесённых из старого `map.html`. Сохраняются исходные поля:

- идентификатор объекта;
- название;
- широта и долгота;
- исходный статус.

Дополнительно для каждого объекта можно заполнить тип, адрес, трансформатор, мощность, ПУ, ТТ, коэффициент трансформации, ПУ освещения, состояние обследования и заметки.

Кнопка «В рабочий проект» переносит выбранную ТП в текущую полевую сессию и открывает её карточку. Кнопка «Импорт map.html» принимает старый HTML-файл или JSON-резервную копию и объединяет записи по ID, не создавая дубликаты.

Исходный файл для автоматической загрузки находится в `public/map-source.html`. При первом запуске «Контур» загружает его и сохраняет нормализованные записи в `localStorage` под ключом `kontur-tp-registry-v1`.

Данные при таком сценарии хранятся только в браузере на вашем устройстве. Это удобно для единоличной работы и не требует сервера, но при очистке данных браузера резервная копия JSON будет единственным способом восстановления. В реестре используйте «Экспорт базы», а для полного проекта — «Экспорт отчёта».

## Логика дерева сети

Данные обследования хранятся локально в браузере с явной иерархией:

```text
КТП
└── Фидер
    └── Опора
        ├── ПУ 1
        └── ПУ 2
```

Каждый ПУ является отдельной записью. Поэтому два прибора на одной опоре выгружаются двумя строками Excel, но остаются дочерними элементами одной опоры.

## Публикация на GitHub Pages

В проект уже добавлен workflow `.github/workflows/deploy.yml`.

1. Создайте репозиторий и загрузите файлы проекта в ветку `main`.
2. В GitHub откройте `Settings → Pages` и выберите `GitHub Actions`.
3. После push workflow соберёт Vite-проект и опубликует папку `dist`.

Ключ Яндекс.Карт не является секретом в статическом приложении, поэтому ограничьте его разрешёнными доменами GitHub Pages.
